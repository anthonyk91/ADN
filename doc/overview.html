Abstract Deep Network


This package includes an implementation of the ADN algorithms, and routines for running experiments based on script files.


The software is written as a template method, where the ADN library is based around a process of making an observation, performing an action, and making updates as necessary. It is written as an OO implementation, with the intention of allowing parts to be modified in a flexible manner. This can be done by subclassing, where parts can be extended or replaced to change the design.


Dependencies:


gsl, boost


Installing


The following is an example of how to get the learner compiled and running.


* edit the file src/makefile-defs, to specify the compiler, include path and library path (for the

dependencies)

* run make (to build the "release" executable, see other targets in makefile)


This will build the file build/Release/adn-point



Running


Command-line method


Simple execution can be done using command line arguments. (This approach expects data to be in .csv format)


Options:

  -s [ --script-file ] arg  script file

  -i [ --script-stdin ]     read script from
stdin

  -n [ --seed-value ] arg   random number seed

  -v [ --version ]          show version

  -t [ --train-file ] arg   load training file
  -r [ --predict-file ] arg load prediction file
  -f [ --format] arg        file format (arff
(def) or csv)
  -m [ --minutes ] arg      training time
(mins)

  -d [ --model ] arg        load model

  -w [ --write-model ] arg  write model

  -o [ --output ] arg       prediction output

  -p [ --showpid ]          add PID to output

The following command performs training on the “short.csv” training data, and stops after 2 minutes, using the “cmdline.script” file to define hyper-parameters.


./adn-point -t ../data/short.csv -f csv -m 2 -s
csv.script 
-w mymodel

And to produce predictions on a test set:


./adn-point -r ../data/test.csv -f csv -s
cmdline.script -d mymodel -o predictions

With a small change to the cmdline.script file predictions can be output as real-values, by changing the entry predictionClasses=1 to 0.

Script based method

The more common method is to define commands in the script file. Action commands to load data and start runs are specified using the “ac:” symbol, and are processed sequentially. (In contrast “key=value” pairs are definitions without order.)


The following sample script file will load training and test data from file in ARFF format, and start an experiment. Most of the parameters are stored in a separate file (common.script).


import common.script



runName = wav





maxEpochs = 400



learningDecay = 0.98



maxComposites = 500



maxAtomFragments = 250





ac: loadARFF ../d/wav/TrainFold0 1



ac: loadARFF ../d/wav/TestFold0 0



ac: run








Overview







For a description of
the algorithm see (preprint link).







The ADN library
(/ADN) contains definitions for fragments (atomic and composite), and
a harness for managing the lifecycle of fragments. The harness has
routines for making an observation of the world, passing activations
through the feature network, and making adjustments after receiving
feedback.







The
following diagram gives a rough overview of classes in ADN and their
relationships.













Implementations
extend these basic classes, as shown below.  The Point folder
(/Point) extends the classes defined in the ADN library, and provides
a working implementation. This operates on environments (datasets)
with a set of real, int or nominal values per instance, typical of
UCI/ARFF formats.  Gradient descent operations have been defined in
subclasses (GradFragment) of various classes (eg PtFragment) in the
Point module, replacing the logical activation methods with
continuous versions, and implementing backprop routines.













(The convolutional
implementation, which operates on 2D inputs such as images, is not
included as it doesn't fit with the code after I reworked the other
parts.)
